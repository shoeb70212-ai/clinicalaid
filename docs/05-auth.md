# Authentication & Security — V1

---

## Role Design

| Role | Portal Access | Data Access | 2FA |
|---|---|---|---|
| doctor | /doctor only | Own session patients only | TOTP mandatory |
| receptionist | /reception only | All clinic patients, no medical notes | TOTP optional |
| admin | All portals | Full clinic data + settings | TOTP mandatory |
| display | /display only | queue_display_sync only — zero PII | None — scoped JWT |

A doctor CANNOT see another doctor's patients.
Enforced by filtering queue_entries by session_id (doctor-specific), not RLS alone.

## JWT Structure

```json
{
  "sub":        "auth-user-uuid",
  "email":      "doctor@clinic.com",
  "role":       "doctor",
  "clinic_id":  "clinic-uuid",
  "staff_id":   "staff-uuid",
  "exp":        1234567890
}
```

`clinic_id` → used by every RLS policy for tenant isolation
`staff_id`  → read by audit trigger — no extra DB lookup needed
`role`      → used by route guards and permission checks

## JWT Enrichment Edge Function

```typescript
// supabase/functions/jwt-enrichment/index.ts
// Runs as Supabase Auth Hook on every login

import { createClient } from '@supabase/supabase-js';

Deno.serve(async (req) => {
  const { user_id } = await req.json();

  const supabase = createClient(
    Deno.env.get('SUPABASE_URL')!,
    Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')!
  );

  const { data: staffRecord, error } = await supabase
    .from('staff')
    .select('id, clinic_id, role, is_active')
    .eq('user_id', user_id)
    .eq('is_active', true)
    .single();

  // Handle multi-clinic staff (V2): .single() throws if multiple rows
  // V1: one doctor = one clinic. If multiple clinics exist, show clinic selector.
  if (error || !staffRecord) {
    return new Response(
      JSON.stringify({ error: 'Staff record not found or inactive' }),
      { status: 403 }
    );
  }

  return new Response(
    JSON.stringify({
      clinic_id: staffRecord.clinic_id,
      staff_id:  staffRecord.id,
      role:      staffRecord.role,
    }),
    { headers: { 'Content-Type': 'application/json' } }
  );
});
```

## Display Role JWT

TV display gets a separately issued, scoped JWT:

```json
{
  "sub":       "display-device-uuid",
  "role":      "display",
  "clinic_id": "clinic-uuid",
  "session_id":"session-uuid",
  "exp":       9999999999
}
```

Generated by admin in clinic settings.
Stored in the TV device browser (localStorage) — never expires in V1 (long exp).
Can be rotated by admin if compromised.
This JWT grants SELECT only on queue_display_sync for the clinic's data.
It cannot read queue_entries, patients, or any other table.

## Route Guards

```typescript
// src/components/ProtectedRoute.tsx
import { Navigate } from 'react-router-dom';
import { useAuth } from '../hooks/useAuth';

interface Props {
  children: React.ReactNode;
  allowedRoles: string[];
  requireMFA?: boolean;
}

export function ProtectedRoute({ children, allowedRoles, requireMFA = false }: Props) {
  const { session, role, mfaVerified, loading } = useAuth();

  if (loading)          return <div>Loading...</div>;
  if (!session)         return <Navigate to="/login" replace />;
  if (!allowedRoles.includes(role)) return <Navigate to="/unauthorized" replace />;
  if (requireMFA && !mfaVerified)   return <Navigate to="/verify-mfa" replace />;

  return <>{children}</>;
}

// Router usage:
// /doctor/*   → allowedRoles={['doctor','admin']} requireMFA={true}
// /reception/* → allowedRoles={['receptionist','admin']}
// /display     → no ProtectedRoute — display JWT handled separately
// /setup       → no ProtectedRoute — unauthenticated
```

## TOTP 2FA

Supabase Auth natively supports TOTP (Google Authenticator, Authy).
Enable in Supabase Dashboard → Authentication → MFA → Enable TOTP.

```typescript
// src/lib/auth.ts
import { supabase } from './supabase';

export async function loginWithMFA(email: string, password: string) {
  const { data, error } = await supabase.auth.signInWithPassword({ email, password });
  if (error) throw error;

  const { data: mfaData } = await supabase.auth.mfa.getAuthenticatorAssuranceLevel();

  if (mfaData?.nextLevel === 'aal2' && mfaData?.currentLevel !== 'aal2') {
    return { requiresMFA: true };
  }
  return { requiresMFA: false, session: data.session };
}

export async function verifyTOTP(code: string) {
  const { data: factors } = await supabase.auth.mfa.listFactors();
  const totpFactor = factors?.totp?.[0];
  if (!totpFactor) throw new Error('No TOTP factor enrolled');

  const { data: challenge } = await supabase.auth.mfa.challenge({ factorId: totpFactor.id });
  if (!challenge) throw new Error('Challenge creation failed');

  const { data, error } = await supabase.auth.mfa.verify({
    factorId: totpFactor.id,
    challengeId: challenge.id,
    code,
  });
  if (error) throw error;
  return data;
}
```

## Inactivity Logout (BroadcastChannel — all tabs)

```typescript
// src/hooks/useInactivityLogout.ts
// Uses BroadcastChannel to sync last-active timestamp across ALL open tabs.
// Logout only fires if ALL tabs have been idle beyond the timeout.
// Prevents Tab B timing out and killing the session active in Tab A.

import { useEffect, useRef } from 'react';
import { supabase } from '../lib/supabase';

const CHANNEL_NAME = 'clinicflow_activity';

export function useInactivityLogout(timeoutMs: number) {
  const timer  = useRef<ReturnType<typeof setTimeout>>();
  const bc     = useRef<BroadcastChannel>();

  const broadcast = () => {
    bc.current?.postMessage({ lastActive: Date.now() });
  };

  const resetTimer = () => {
    clearTimeout(timer.current);
    broadcast();
    timer.current = setTimeout(async () => {
      await supabase.auth.signOut();
      window.location.href = '/login';
    }, timeoutMs);
  };

  useEffect(() => {
    bc.current = new BroadcastChannel(CHANNEL_NAME);

    // If another tab is active, reset our timer
    bc.current.onmessage = () => {
      clearTimeout(timer.current);
      timer.current = setTimeout(async () => {
        await supabase.auth.signOut();
        window.location.href = '/login';
      }, timeoutMs);
    };

    // Throttle event listeners to fire max once per second
    let lastFired = 0;
    const throttledReset = () => {
      const now = Date.now();
      if (now - lastFired > 1000) {
        lastFired = now;
        resetTimer();
      }
    };

    const events = ['mousedown','keydown','touchstart','scroll'];
    events.forEach(e => window.addEventListener(e, throttledReset));
    resetTimer();

    return () => {
      events.forEach(e => window.removeEventListener(e, throttledReset));
      clearTimeout(timer.current);
      bc.current?.close();
    };
  }, [timeoutMs]);
}

// Usage:
// DoctorPortal:    useInactivityLogout(15 * 60 * 1000)  // 15 minutes
// ReceptionPortal: useInactivityLogout(30 * 60 * 1000)  // 30 minutes
```

## Session Timeout Rules

| Context | Timeout | Behaviour |
|---|---|---|
| Doctor portal | 15 min idle | Auto-logout via BroadcastChannel |
| Reception portal | 30 min idle | Auto-logout via BroadcastChannel |
| TV display | Never | Stays active indefinitely |
| JWT access token | 1 hour | Auto-refreshed silently by Supabase SDK |
| JWT refresh token | 7 days | User must re-login after 7 days |
| Display JWT | Long-lived | Admin rotates manually if compromised |

## Staff Invite Flow

```typescript
// 1. Admin generates invite:
// INSERT INTO staff_invites(clinic_id, email, role, token, created_by)
// token = crypto.randomUUID() + timestamp hash (secure random, not predictable)

// 2. Admin sends link to receptionist via WhatsApp/email:
// https://app.clinicflow.in/invite?token=<token>

// 3. Receptionist opens link:
// - System validates: token exists, not used, not expired
// - System pre-fills email from invite record
// - Receptionist sets password (email already locked)
// - On submit: Supabase Auth creates user, staff row created with clinic_id from invite
// - staff_invites.used_at = NOW()
// - JWT enrichment fires on first login — injects clinic_id + role

// 4. Invite expiry: 48 hours. Single use. Cannot be reused after used_at is set.
```

## Security Rules Summary

| Rule | Implementation |
|---|---|
| HTTPS mandatory | Supabase Managed enforces TLS — no config needed |
| CORS: own domain only | Supabase project settings — restrict allowed origins |
| Login lockout | Max 5 failed attempts → 15-minute lockout (Supabase Auth setting) |
| PII in error logs | Sentry config: scrub name, mobile, dob fields from all payloads |
| Display portal PII | Display JWT grants SELECT on queue_display_sync only — no other table |
| Fired staff | `is_active = FALSE` → RLS blocks next request — no wait for JWT expiry |
| Signature security | Doctor SVG signature stored encrypted (V2: decrypted only at PDF render) |
